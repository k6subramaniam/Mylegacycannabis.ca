Title: 🧹 Fix robots.txt rules for public endpoints

🎯 What:
Updated the robots.txt rules in `server/_core/index.ts` to keep `Disallow: /api/` as the default to block all APIs from crawlers, while explicitly allowing specific public store endpoints and the `nearest-store` endpoint needed by Googlebot to render the SPA.

💡 Why:
The previous iteration removed the generic `/api/` disallow rule which could potentially expose other APIs that shouldn't be crawled. The correct approach is to keep the default `Disallow: /api/` rule to block everything, but place specific `Allow:` rules *above* it for the endpoints Googlebot actually needs to fetch data for product listings. More specific rules take precedence in robots.txt.

✅ Verification:
- Looked at `git diff` to confirm that `Allow:` rules are placed above `Disallow: /api/`.
- Tests passed locally.

✨ Result:
All APIs will be blocked by default while specific store data endpoints needed for rendering product listings will be available to Googlebot.
