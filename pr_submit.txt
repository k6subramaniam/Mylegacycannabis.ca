Title: 🧹 Fix robots.txt rules for public endpoints

🎯 What:
Updated the robots.txt rules in `server/_core/index.ts` to allow search engines to crawl public store endpoints (`/api/trpc/store.*`) and geo endpoints (`/api/geo/`), while explicitly blocking sensitive paths like admin and authentication endpoints.

💡 Why:
The previous robots.txt rule indiscriminately blocked `/api/`, preventing search engines from accessing public data while also not explicitly specifying which TRPC routes should be excluded.

✅ Verification:
- Looked at `git diff` to confirm that `Allow: /api/trpc/store.` and `Allow: /api/geo/` are added, and `/api/trpc/admin.`, `/api/trpc/auth.`, and `/api/auth/` are disallowed.
- Note regarding the database migration requirement: The required `ALTER TABLE users ADD COLUMN address_*` SQL commands are already implemented in `server/db.ts` within the `initializeDb` function and run automatically upon application deployment to Railway, making an explicit migration script unnecessary.
- Tests passed locally.

✨ Result:
Search engine crawlers will now be able to successfully crawl public application data.
